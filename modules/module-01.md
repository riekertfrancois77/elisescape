# Module 1 — Think Like a Game Studio

**Status:** UNLOCKED
**Role:** Eli is Creative Director. Claude Code is the studio team.

## Purpose

Design your own game from scratch — before a single line of game code is
written. Real studios don't start with code. They start with *decisions*.
This module ends with a design so clear the whole team (human and AI) knows
exactly what game we're making, and why.

## What You'll Learn

- AI helps with **thinking**, not just coding.
- Great games begin with **clear design choices**.
- The human is the **Creative Director** — you.
- AI can act as a **studio team**: designers, writers, engineers, critics.
- The project memory files become the **shared brain** of the studio.
- Good prompts create better outcomes.

## Rules for Claude Code in This Module

- Never suggest a theme, genre, setting, story, or mechanic before Eli has
  answered the questions. His answers are the only raw material.
- Ask one question at a time. Dig deeper on interesting answers.
- Record every question and answer in `logs/questions-eli-answered.md`.
- Record locked decisions in `logs/decisions-made.md`.
- Update `MEMORY.md` (Module 1 step checkboxes, Session Memory, Claude
  Instructions) as steps complete.

## Mission Flow

Work through these steps in order. Each step is a conversation, not a form.

1. **Welcome to the Studio** — Claude welcomes Eli as Creative Director,
   explains how the studio works, and how the memory files work.
2. **What should players feel?** — Excitement? Fear? Curiosity? Power?
   Feelings come before features.
3. **Who does the player become?** — Not "what character" — who does the
   *player* get to be while playing?
4. **What kind of world is this?** — Big or small, real or impossible,
   friendly or dangerous. Eli describes it; Claude asks follow-ups.
5. **What problem or challenge drives the game?** — Every great game has an
   engine of tension. What's ours?
6. **What are the game's 3–5 pillars?** — The short list of things this game
   *is*. Every future decision gets tested against them.
7. **What references inspire the game?** — Games, movies, books, places,
   anything. Steal like a director.
8. **Three game directions** — Claude generates 3 possible game directions
   built **only** from Eli's answers above. No outside ideas.
9. **The Decision** — Eli chooses one, rejects them, or combines pieces.
   This is the First Big Decision. It goes in `logs/decisions-made.md`.
10. **Studio team review** — Claude simulates a studio review: a designer, an
    engineer, an artist, and a skeptical producer each react to the chosen
    direction. Eli responds as Director. The design gets sharper.
11. **Write the design harness** — Claude fills in the `design/` files from
    everything decided: vision, pillars, player experience, story & world,
    mechanics, art direction, technical direction, future ideas.
12. **Achievement unlock: Creative Director** — Claude marks the achievement
    in `ACHIEVEMENTS.md`, checks Module 1 in `MEMORY.md`, and tells Eli to
    refresh the dashboard.

## Done When

- All `design/` files are filled in from Eli's decisions.
- `logs/` has the full paper trail.
- The **Creative Director** achievement is unlocked.
- Eli can say what his game is in one sentence — and mean it.
